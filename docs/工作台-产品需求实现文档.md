# 工作台 · 产品需求实现文档

本文档基于当前工程代码整理，描述 Aqara Builder「工作台」模块的入口、Tab 结构、空间建模、健康度、逻辑设计与调试诊断等能力。

---

## 1. 模块概述

**工作台**（路径 `/lab/`，可选查询参数 `?studio=xxx`）是 Builder 的编辑与运行中枢，包含四个 Tab：**空间**、**健康度**、**逻辑设计**、**调试诊断**。支持从总览「进入实验室」跳转（带 `studio` 时全屏无壳）；顶栏展示当前目标 Studio 或下发目标选择，以及「远程连接 Local Studio」入口。

---

## 2. 路由与布局

### 2.1 路径与参数

- `/lab`：无指定 Studio；下发目标为「不指定（对所有 Studio）」或通过下拉选择某 Studio（会跳转到 `/lab?studio=xxx`）。
- `/lab?studio=<id>`：当前目标为指定 Studio；且由 `ConditionalShell` 判断不渲染 AppShell，整页全屏工作台。

### 2.2 顶栏（工作台内）

- **全屏时**：左侧「← 返回」链接到 `/spaces/`。
- **当前目标**：若有 `studio`，显示「当前目标：{studio.name}」+ 按钮「远程连接 Local Studio」（新开窗口打开 `getStudioWebUrl(studioId)`）。
- **无 studio 时**：若有可选 Studio 列表，显示「下发目标：」下拉框，选项为「不指定（对所有 Studio）」+ 各 Studio；选择后 `router.push(/lab?studio=xxx)` 或 `/lab`。
- **无任何 Studio**：文案「未指定 Studio」。

### 2.3 Tab 栏

- 四个 Tab：**空间**、**健康度**、**逻辑设计**、**调试诊断**；默认 `activeTab = "space"`。
- 点击切换 Tab，内容区渲染对应视图组件。

---

## 3. 空间 Tab（LabSpaceView）

### 3.1 目标

- 构建空间本体（户型、房间、设备/家具），现场实施时可同步到本地 Studio。

### 3.2 顶条

- 说明：「构建空间本体，现场实施时可同步到本地 Studio」；若有 `studioName` 显示「· 当前目标：{studioName}」。
- 操作按钮：同步、+ 新增实体、保存（当前为占位）。

### 3.3 左侧 · 空间结构

- 标题「空间结构」；搜索框 placeholder「搜索空间/设备/家具」。
- 筛选：全部 / 已配置图元 / 未配置图元（`filter`）。
- 树形结构：房间（可展开/收起），子节点为设备（⊙）或家具（▢）；Mock 数据 `MOCK_SPACE_STRUCTURE`（客厅、主卧、厨房、卫生间及各自 children）。
- 底部统计：「19 个对象 · 8 条关系」。

### 3.4 中间 · 组件库

- 说明：「拖拽图标到户型图中进行配置」。
- 分类：家具（沙发、床、桌子、椅子、电视柜）、设备（吸顶灯、传感器、空调、窗帘）；每项可拖拽（draggable）。

### 3.5 右侧 · 户型视图

- 标题「户型视图」；图层切换：房间、天花板、地板；操作「方位关系生成」「重新上传识别」。
- 主区域：虚线边框的占位画布（min-h-320px），用于放置从组件库拖入的实体。

---

## 4. 健康度 Tab（LabHealthView）

### 4.1 单 Studio / 多 Studio

- **单 Studio**：URL 带 `?studio=xxx` 时，仅展示该 Studio 的一张健康卡片。
- **多 Studio**：无 `studio` 时，展示传入的 `studios` 列表，每台一张卡片。

### 4.2 健康卡片内容（StudioHealthCard）

- 标题行：Studio 名称；标签：型号（M300 / M200 网关）、状态（正常/降级/离线，颜色区分）。
- 位置：`studio.location`。
- 指标网格：CPU 负载、内存、最后心跳、运行时长、Logic 成功率、Graph 节点、网络延迟（若有）、设备在线率（若有）、固件。
- 运行时长由 `formatUptime(studio.uptime)` 格式化为「x 天 x 时 x 分」。

---

## 5. 逻辑设计 Tab（LogicDesignView + LogicBlockEditor）

### 5.1 设备面板（未选 Studio 时）

- 仅当 `!studioId` 时显示左侧设备面板。
- 说明：「勾选或拖拽到画布参与编排，下发时按 Studio 分组。」
- 按 Studio 分组展示设备列表（Mock：`MOCK_DEVICES_BY_STUDIO`）；每设备可勾选、可拖拽；暂无 Studio 时提示「暂无 Studio，请先在总览纳管。」
- 底部入口：「按需编排（轻量设备）」→ 进入按需编排简页。

### 5.2 已选 Studio 时

- 不显示设备面板，仅画布区；画布上方有链接「按需编排」。

### 5.3 按需编排（独立简页）

- 顶部：「按需编程」标题 +「返回逻辑设计」。
- 说明：「不支持完整 Logic 块的设备，可在 Builder 上按需编排并下发，由平台做协议转换与推送。」
- 按钮：「新建按需编排」（当前为占位）。

### 5.4 逻辑块编辑器（LogicBlockEditor）

- **顶栏**：可选展示当前目标/下发选择、远程连接 Local Studio（与工作台顶栏一致时由外层传入 `showTargetBar=false`）；备注图标、缩放（−/+/ 百分比）、启用开关、属性面板开关；流程提示「编排 → 编译 → 下发」；**编译**按钮（点击后 `setCompileOk(true)`）；**下发到设备**按钮（仅当 `compileOk` 为 true 时可点）。
- **左侧 · 逻辑块面板**：分类（变量、逻辑卡片、算术运算函数、时间、其它）；每类下为具体块（如布尔变量、与、等于、定时、脚本卡片等）；搜索框「搜索块」。
- **中间 · 画布**：网格背景；缩放比例作用于画布；Mock 放置若干 `CanvasBlock`（如布尔变量、脚本卡片、大于、小于等于等），带输入/输出连接点样式。
- **右侧 · 属性面板**：`showProps` 为 true 时显示；说明「选中逻辑块后可在此编辑参数」（当前为占位）。

---

## 6. 调试诊断 Tab（LabDebugView）

### 6.1 说明

- 标题「调试诊断」；若有 `studioName` 显示「当前目标：{studioName} · 查看实时日志、执行远程命令」；否则「选择 Studio 后可查看该实例的日志与执行调试命令」。

### 6.2 日志区

- 固定高度、可滚动；Mock 日志行：时间、[level]、消息内容；level 为 error/warn 时使用不同颜色（如 text-red-400、text-amber-400）。
- Mock 示例：心跳上报、Logic 执行完成、Device 信号弱、Auth 会话、Logic 执行超时等。

### 6.3 命令输入

- 输入框 placeholder：「输入命令，如 status / restart_logic / ...」；右侧「执行」按钮（当前为占位）。

---

## 7. 数据与 Mock

### 7.1 Studio 与 URL

- `STUDIOS_BY_ID`、`getAllStudioIds()` 来自 `lib/studios-mock.ts`。
- `getStudioWebUrl(studioId)`：基于 `MOCK_STUDIO_WEB_BASE` 与 `MOCK_STUDIO_AUTH_TOKEN` 生成远程 Local Studio 登录链接。

### 7.2 设备分组（逻辑设计）

- `MOCK_DEVICES_BY_STUDIO`：按 studioId/studioName 分组，每组下 devices 含 id、name、type。

### 7.3 空间结构（空间 Tab）

- `MOCK_SPACE_STRUCTURE`：房间 id、name、type: "room"，children 为设备/家具 id、name、type。
