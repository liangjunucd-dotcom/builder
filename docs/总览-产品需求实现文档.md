# 总览 · 产品需求实现文档

本文档基于当前工程代码整理，描述 Aqara Builder「总览」模块的页面结构、视角切换、项目与站点、Studio 运行控制及地图等能力。

---

## 1. 模块概述

**总览**（路径 `/spaces/`）是 Builder 的实例与项目中枢：支持两种视角——**业务协作**（Space → Project → Site → Studio）与 **运行控制**（Studio 列表与地图）。用户可在此创建/编辑项目与站点、将 Studio 注册到当前 Builder Space、在列表或地图上查看 Studio 状态、打开 Studio 详情抽屉并进入工作台。

---

## 2. 页面结构

### 2.1 顶区（Header）

- **标题**：`总览`
- **副标题**：随视角切换显示
  - 业务协作：`项目与站点`
  - 运行控制：`Studio 运行状态`
- **视角切换**：两个按钮
  - `业务协作`（perspective = collab）
  - `运行控制`（perspective = runtime）
- **Builder Space 选择器**：下拉 `<select>` 绑定 `currentBuilderSpace?.id`，选项为 `builderSpaces`；切换时 `setCurrentBuilderSpaceId` 并清空 `selectedStudio`。
- **运行控制视角下额外控件**：列表/地图视图切换（图标按钮：列表 | 地图），仅当 `perspective === "runtime"` 时展示。

### 2.2 新手引导（可关闭）

- 条件：`!hideGuide`，且从 localStorage 读取 `builder_hide_spaces_guide`，若为 `"true"` 则不再显示。
- 内容：简短说明「首次使用？单击行可查看详情，点击「连接」进入 Studio 工作台」；链接到首页；右侧按钮「不再显示」写入 localStorage 并设置 `hideGuide = true`。

### 2.3 无 Space 时的空状态

- 条件：`!currentBuilderSpace`。
- 文案：「请先选择或创建 Builder Space」；引导链接「在顶栏切换 Space，或前往 **认证与组织** 创建」指向 `/identity/`。

## 3. 业务协作视角（collab）

### 3.1 数据与筛选

- **项目来源**：`getProjectsByBuilderSpaceId(currentBuilderSpace.id)`，仅当前 Space 下的项目。
- **筛选**：
  - 建筑类型（BuildingType）：Office / Home / Stadium / Hotel / School / Other，存于 `filterType`。
  - 国家：ISO 3166-1 alpha-2（如 CN、US、JP 等 + 显示名），存于 `filterCountry`。
- **清除筛选**：`filterType`、`filterCountry` 置空；有任一筛选时显示「清除筛选」按钮。
- **视图**：`projectView` 为 `list`（表格）或 `card`（卡片网格）。

### 3.2 工具栏

- 左侧：列表/卡片切换、Type 下拉、Country 下拉、清除筛选（有筛选时）。
- 右侧：「创建项目」按钮，打开创建项目弹窗。

### 3.3 项目列表（表格）

- 表头：Project Name、Type、Studios、Country、Updated On、Created On、Devices、操作。
- **Studios**：项目下所有站点关联的 Studio 实例数量（通过 `getProjectStudioAndDeviceCount` 计算）。
- **Devices**：上述 Studio 对应的 `graphNodes` 汇总。
- 每行可展开（`expandedProjectId`）：展开后显示该项目下所有**站点**；每个站点卡片含：
  - 站点名称、编辑、关联 Studio；
  - 已关联的 Studio Cloud 实例列表（实例名 → Local Studio 名）、「连接」跳转工作台、解除关联（×）；
  - 无实例时显示「未绑定」；
  - 底部「+ 添加站点」。
- 行内操作：「编辑」打开编辑项目抽屉；「站点」打开添加站点弹窗（并带当前 projectId）。

### 3.4 项目卡片（卡片视图）

- 卡片内容：项目名称、Type、Studios、Country、Updated、Created、Devices。
- 操作：「编辑」「管理站点」（切换回列表并展开该项目）。

### 3.5 创建项目

- 弹窗字段：项目名称（必填）、国家（选填）、建筑类型（选填）、客户/交付对象（选填）。
- 提交：`createProject({ builderSpaceId, name, country?, buildingType?, clientName? })`，关闭弹窗并清空表单。

### 3.6 编辑项目（侧边抽屉）

- 右侧抽屉（小屏全宽，大屏约 28rem–32rem），可滚动。
- 表单：项目名称、国家、建筑类型、客户/交付对象。
- **协作者**：输入邮箱 +「添加」调用 `addProjectCollaborator(editingProjectId, email)`；列表展示当前协作者及角色，可「移除」`removeProjectCollaborator`。
- 底部：复制项目（`duplicateProject`）、移除项目（二次确认）、取消、保存（`updateProject`）。
- 移除项目二次确认：说明「移除后该项目及其下所有站点与 Studio 关联将被删除，且无法恢复」。

### 3.7 站点：创建/编辑

- 弹窗：「添加站点」或「编辑站点」；字段：站点名称、地址/备注（选填）。
- 提交：`createSite({ projectId, name, address? })` 或 `updateSite(siteId, { name, address? })`。

### 3.8 关联 Studio 到站点

- 条件：从已**注册**到当前 Space 且**尚未关联到任何站点**的 Local Studio 中选择（`unboundRegisteredIds`）。
- 弹窗：可选「显示名称」；列表为 `unboundRegisteredIds` 对应的 Studio（名称 + id）；点击某项即 `bindStudioToSite(siteId, localStudioId, displayName?)`。
- 若无可用 Studio：提示「暂无已注册 Studio。Local Studio 在本地登录 Builder 账号后选择「Builder Space」绑定即完成注册。」或「当前无可关联的 Studio，已注册的 Studio 均已关联到站点。」

### 3.9 解除 Studio 与站点关联

- 站点卡片内点击实例旁的「×」→ 二次确认弹窗 → `unbindStudioFromSite(instanceId)`。

## 4. 运行控制视角（runtime）

### 4.1 数据来源

- **Studio 列表**：  
  - 业务协作视角下用于「项目/站点」的 Studio 来自当前 Space 的层级或 `currentBuilderSpace.studioIds`（`localStudioIds` → `getStudiosByIds` 得到 `nodesForFleet`）。  
  - 运行控制视角下列表与地图使用 `getAllStudioIds()` 再 `getStudiosByIds` 得到 `nodesForRuntimeList`，即 `nodes`。
- **Fleet 统计**：`computeFleetStats(nodesForFleet)` 得到 totalNodes、onlineNodes、degradedNodes、offlineNodes、avgCpu、totalGraphNodes、totalLogicCredits、aiInferenceCalls。

### 4.2 视图切换：列表 / 地图

- 顶栏或内容区上方：按钮「列表」「地图」，`view === "list"` 或 `view === "map"`。
- 列表：表格；地图：先展示 Fleet 概览卡片区，再展示 NodeMapView。

### 4.3 运行控制列表（表格）

- 列：设备名称、Device ID、IP、最后可见、注册到平台、操作。
- **设备名称**：Studio 名称 + 当前 Builder Space 名称（小字）。
- **Device ID**：`node.id`，带复制按钮。
- **IP**：Mock 由 `node.id` 生成（如 10.11.x.x），带复制按钮。
- **最后可见**：`node.lastHeartbeat`。
- **注册到平台**：  
  - 若 `registeredIds.includes(node.id)`：显示「已注册」+「取消」按钮（`unregisterLocalStudio(spaceId, node.id)`）。  
  - 否则：「注册到本 Space」按钮（`registerLocalStudio(spaceId, node.id)`）。
- **操作**：「连接」调用 `handleDrillDown(node)` → `recordRecentOpened({ type: "lab", id, label, href: /lab?studio=... })` 并 `router.push(/lab?studio=...)`。
- 行点击：选中/取消选中该 Studio，用于打开详情抽屉（`setSelectedStudio`）。
- 底部文案：`{nodes.length} 个实例`。

### 4.4 运行控制地图

- **FleetOverview**：展示 8 个统计块（Total Nodes、Online、Degraded、Offline、Avg CPU、Graph Nodes、Logic Credits、AI Calls）。
- **NodeMapView**：基于 OpenStreetMap（Leaflet），传入 `nodes`、`selectedId`、`onSelect`、`onDrillDown`；支持点击选中、双击进入实验室（`onDrillDown`）；图例区分运行正常/降级/离线。
- Studio 节点需包含 `lat`、`lng`（来自 `studios-mock`）。

### 4.5 无 Studio 时的空状态

- 条件：已选 Space 且 `nodes.length === 0`。
- 文案：「当前 Space 下暂无 Studio 实例，请绑定或切换 Space。」

## 5. Studio 详情抽屉（StudioDetailDrawer）

### 5.1 触发与关闭

- 运行控制视角下，在列表或地图中**选中**某 Studio 时打开；`selectedStudio` 有值即渲染抽屉。
- 关闭：点击遮罩、点击关闭按钮、按 Escape；回调 `onClose` 即 `setSelectedStudio(null)`。

### 5.2 布局与内容

- 右侧固定宽度抽屉（max-w-md），带遮罩；使用 `createPortal` 挂到 `document.body`。
- **头部**：Studio 名称、位置（带图标）；关闭按钮。
- **状态概览**：  
  - 状态标签：运行正常 / 降级 / 离线（颜色区分）。  
  - 若有 `node.model`：M300 完整 Studio / M200 网关。  
  - 心跳、固件。  
  - M200 时说明：「M200 等网关已连接用户 AIOT 平台，Builder 可通过 Studio Cloud 从 AIOT 拉取数据同步。」
- **健康度监控**：CPU 负载、内存、网络延迟（若有）、设备在线率（若有）、Logic 成功率、Graph 节点数、空间数（若有）、运行时长。
- **需求预警**：`relatedAlerts`（与当前 Studio 匹配的 MOCK_ALERTS），展示 message、since。
- **操作**：  
  - 「进入实验室」：`onGoToLab(node)` → 同 `handleGoToLab`，先关抽屉再跳转 `/lab?studio=xxx`。  
  - 「查看 Live Graph」：占位。  
  - 说明：「深度下钻：在列表或地图中双击该节点，可进入该子空间的内部逻辑视图。」

### 5.3 告警 Mock

- `MOCK_ALERTS`：含 id、studioId、space、type、message、level、since；与 `selectedStudio.id` 匹配后传入抽屉的 `relatedAlerts`。

## 6. 数据与类型

### 6.1 项目与站点（ProjectsContext）

- `getProjectsByBuilderSpaceId`、`getSitesByProjectId`、`getStudioCloudInstancesBySiteId`、`getLocalStudioIdsForSpace`、`getRegisteredLocalStudioIds`；
- `createProject`、`updateProject`、`duplicateProject`、`deleteProject`；
- `addProjectCollaborator`、`removeProjectCollaborator`；
- `createSite`、`updateSite`、`deleteSite`；
- `bindStudioToSite`、`unbindStudioFromSite`；
- `registerLocalStudio`、`unregisterLocalStudio`。

### 6.2 Studio 与 Mock

- `lib/studios-mock.ts`：`STUDIOS_BY_ID`、`getStudiosByIds`、`getAllStudioIds`。
- `StudioNode`（ClusterCard）：id、name、location、status、cpuLoad、memoryUsage、logicBlocksTotal/Success、uptime、lastHeartbeat、graphNodes、firmware、lat、lng、networkLatencyMs、deviceOnlineRate、spacesCount、model（M300/M200）。

### 6.3 建筑类型与国家

- `BuildingType`：Office、Home、Stadium、Hotel、School、Other（`lib/domain-types`）。
- `COUNTRY_OPTIONS`：ISO 代码 + 显示名（中国、美国、日本等），用于项目创建/编辑与筛选。
