# 资产库 · 产品需求实现文档

本文档基于当前工程代码整理，描述 Aqara Builder「资产库」模块的 Tab 结构、插件广场、发布方案、我的发布、版本管理及插件详情页等能力。

---

## 1. 模块概述

**资产库**（路径 `/assets/`）面向「空间智能资产化」：将「对人的理解」转化为可订阅的数字资产（SAP）；提供插件广场（方案、插件与驱动）、发布方案（本地上传 / 从 Studio 拉取）、我的发布与版本管理，与硬件兼容产品及订阅分成形成共利闭环。

---

## 2. 页面结构

### 2.1 顶区与 Tab

- **副标题**：「开发者社区 · 资产」
- **标题**：「资产库」
- **说明**：空间智能资产化 · 逻辑提取、方案市场与版本管理；下方卡片强调「信任资产化」与 SAP、插件广场、逻辑提取、脱敏封装、版本回滚及共利闭环。
- **Tab 切换**：四个 Tab——**插件广场**、**发布方案**、**我的发布**、**版本管理**；支持 hash 跳转：`/assets#plugins`、`/assets#extractor`、`/assets#mine`、`/assets#versions`，页面加载与 hash 变化时自动切换 Tab。

---

## 3. 插件广场 Tab

### 3.1 说明

- 文案：插件广场提供方案、插件与驱动一览；仅展示**审核通过**的方案；支持官方/开发者发布，免费或付费订阅，兼容产品与更新日志可见。

### 3.2 插件卡片列表

- 数据源：`MOCK_PLUGINS`（`lib/plugins-mock.ts`）；网格布局（md:2 列、lg:3 列）。
- 每张卡片：封面图（aspect-video）、名称、版本、官方/免费或付费标签、tags、下载数/点赞数、「获取插件」按钮；点击整卡跳转 `/assets/plugin/{id}/`。

### 3.3 插件项类型（PluginItem）

- id、name、version、fileSize、isOfficial、isFree、authorName、authorAvatar、downloadCount、likeCount、description、coverImage、secondaryImage、adaptedProducts、changelog、tags。

---

## 4. 发布方案 Tab

### 4.1 发布方式选择

- 两种方式：**Builder 本地上传**、**从 Studio 拉取**；通过 `publishMethod` 状态切换，对应渲染 `UploadSolutionForm` 或 `SolutionExtractor`。
- 说明：支持两种方式创建方案并提交审核，**审核通过后**才会在插件市场中展示。
- 列表说明：
  - **本地上传**：在 Builder 平台上传已打包的 .sap / .zip，填写版本、类型与简介后提交审核。
  - **从 Studio 拉取**：选择总览中的本地 Studio，系统拉取拓扑、脱敏并映射为标准本体，生成 SAP 后保存草稿或提交审核。

### 4.2 本地上传（UploadSolutionForm）

- **上传方案包**：支持 .sap 或 .zip，单文件不超过 50MB；拖拽或点击选择文件，选中后显示文件名。
- **方案信息**：方案名称、版本号（默认 v1.0.0）、类型（方案/插件/驱动，对应 `SolutionType`）、简介（多行）；底部「保存为草稿」「提交审核」按钮（当前为模拟提交，无真实接口）。

### 4.3 从 Studio 拉取（SolutionExtractor）

- **数据**：Workspace / Studio Mock（与空间站一致）；选择 Studio 后展示该 Studio 的拓扑与映射流程。
- **本体映射**：展示 Graph 节点与标准本体（OntologyAsset）的映射表（MappingEntry：source、target、confidence、status）；支持 mapped / pending / conflict 等状态；可编辑或确认映射。
- **流程**：选择 Studio → 拉取拓扑 → 脱敏与映射 → 生成 SAP → 保存草稿或提交审核（具体步骤以代码实现为准）。

## 5. 我的发布 Tab

### 5.1 说明

- 管理已提交的方案：版本号、类型、状态；仅**已上架**方案会在插件市场中展示；待审核与已拒绝可查看状态或重新提交。

### 5.2 表格列

- 名称、版本、类型、状态、来源、更新时间、操作。
- **类型**：方案 / 插件 / 驱动（`SOLUTION_TYPE_LABELS`）。
- **状态**：草稿 / 待审核 / 已上架 / 已拒绝（`SOLUTION_STATUS_LABELS`）；状态样式区分（灰、琥珀、绿、红）；已拒绝时可显示 `rejectReason`。
- **来源**：本地上传 或 Studio · {studioName}。

### 5.3 操作

- 草稿：编辑、提交审核。
- 已拒绝：修改后重提。
- 已上架：链接「在市场中查看」跳转 `/assets/#plugins`。

### 5.4 数据（SolutionDraft）

- `lib/solution-drafts.ts`：SolutionDraft 含 id、name、version、type、status、source、studioName、updatedAt、rejectReason；`MOCK_MY_SOLUTIONS` 为我的发布 Mock 列表。

---

## 6. 版本管理 Tab

### 6.1 说明

- 基于当前空间逻辑（含提取映射结果）的每一次发布都会生成版本记录；支持一键回滚到历史版本，便于在调优与稳定性之间切换。

### 6.2 版本列表

- Mock：`MOCK_VERSIONS`，每项含 id、label（如 v1.2.0）、time、note。
- 每行：版本号 + 备注、时间、「回滚」按钮（当前为占位）。

---

## 7. 插件详情页（/assets/plugin/[id]/）

### 7.1 路由与数据

- 路径：`/assets/plugin/[id]/`；静态参数由 `generateStaticParams` 从 `MOCK_PLUGINS` 生成。
- 组件：`PluginDetail`，通过 `getPluginById(id)` 获取插件；未找到时显示「未找到该插件」+ 返回资产库链接。

### 7.2 页面内容

- **返回**：按钮「← 返回」调用 `router.back()`。
- **头部卡片**：左侧——插件名称、版本/大小、官方/免费或付费标签、作者头像与名称、「获取插件」按钮、下载数/点赞数/分享；右侧——封面图。
- **描述**：插件 description；若有 secondaryImage 则展示。
- **兼容产品**：`adaptedProducts` 列表，每项名称、价格、链接样式。
- **更新日志**：`changelog` 按版本列出，每版本含 version、fileSize、updated、items（文本列表）。
