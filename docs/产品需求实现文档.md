# Aqara Builder 产品需求实现文档

本文档基于当前工程代码整理，作为产品需求与实现对照说明，便于产品、设计与开发对齐。

---

## 1. 产品概述

**Aqara Builder** 面向空间智能开发者，提供「一人一面 · 空间智能 · 共利」的协作与运行平台。核心能力包括：

- **账号与组织**：Account 为唯一操作者身份；Builder Space 为业务归属主体（项目、资产、Studio 归属）。
- **总览与运行控制**：按 Builder Space 纳管本地或云端 Studio，支持项目/站点维度与运行控制视角（列表 + 地图）。
- **工作台**：空间建模、健康度监控、逻辑设计（编排 → 编译 → 下发）、调试诊断。
- **资产库**：方案/插件资产化、逻辑提取、发布与版本管理。
- **星球**：社区动态、资讯、消息、项目小组协作。
- **信任值**：欢喜度看板、结算中心（SAP 订阅、代维、硬件返点等）。

## 2. 账号与空间体系

### 2.1 规则（见 `lib/account-types.ts`）

| 规则 | 说明 |
|------|------|
| 1 | Account 是唯一真实操作者身份，所有行为与审计追溯到 Account |
| 2 | Builder Space 是业务归属主体（项目、客户、资产、Studio 绑定归属） |
| 3 | Project 是交付与协作的核心单元（首期可简化） |
| 4 | Studio 是运行节点，不是主体 |
| 5 | Aqara 平台拥有全局可见性，单独治理后台，不混入 Builder Space 角色 |

### 2.2 核心类型

- **Account**：`id`、`name`、`email`。
- **BuilderSpace**：`id`、`name`、`kind`（personal | team | company）、`plan`、`members`、`studioIds`、`projectIds`、`region`（CN | US | EU，数据中心）、`companyProfile`、`parentCompanyId`、`createdAt`。
- **BuilderSpaceKind**：个人 / 团队 / 公司。
- **BuilderSpaceRole**：owner | admin | member | viewer。
- **DataCenterRegion**：CN | US | EU，对应 Studio Cloud 部署地域，与合规相关。

### 2.3 当前 Space 与数据中心展示

- 顶栏保留 **Builder Space 选择器**，展示当前 Space 名称及 **数据中心**（如「中国」「美国」「欧洲」）。
- 下拉列表中每个 Space 展示：数据中心 · 类型 · Studio 数。
- 默认首次登录状态包含 `region: "CN"`，并在 `AccountContext` 中持久化。

## 3. 全局导航与布局

### 3.1 顶栏

- **Aqara Builder** 品牌 + Beta 标识。
- **当前 Builder Space** 下拉：显示 Space 名称 + 数据中心（如「· 中国」）；下拉项展示「数据中心 · 类型 · Studio 数」。
- **我的角色**、**快捷入口**（工作台等）在顶栏右侧区域。
- 不展示「当前页标题」（如总览、工作台等），避免与左侧导航重复。

### 3.2 左侧主导航（mainNav）

| 标签 | 路径 | 说明 |
|------|------|------|
| 总览 | `/spaces/` | 实例与项目、Studio 运行状态、地图 |
| 工作台 | `/lab/` | 空间 / 健康度 / 逻辑设计 / 调试诊断 |
| 资产库 | `/assets/` | 插件广场、发布方案、我的发布、版本管理 |
| 星球 | `/collab/` | 动态、资讯、消息、Find Squad |
| 信任值 | `/trust/` | 欢喜度看板、结算中心 |

另有学院导航（论坛、支持）、官方导航（Aqara 商城、官网）等。

### 3.3 工作台全屏模式

- 从总览点击某台 Studio 进入「进入实验室」时，跳转 `/lab?studio=xxx`。
- 当 URL 带 `?studio=` 时，**不渲染顶栏与侧栏**（`ConditionalShell`），整页为全屏 Web 编辑/工作台界面。
- 全屏时顶栏仅保留「← 返回」（回到总览 `/spaces/`）。

## 4. 总览（/spaces/）

### 4.1 视角切换

- **项目与站点**（collab）：Space → Project → Site → Studio 关联；侧重业务协作与交付结构。
- **Studio 运行状态**（runtime）：以 Studio 实例与运行控制为主，列表 + 地图。

### 4.2 主要能力

- **Builder Space 选择**：切换当前 Space，筛选该 Space 下的项目、站点与 Studio。
- **项目与站点**：创建/编辑项目（名称、建筑类型、国家等）；项目下站点；站点关联 Studio（从已注册且未关联的 Studio 中选择）。
- **运行控制列表**：设备名称、Device ID、IP、最后可见、绑定时间、操作；支持管理从local studio绑定到当前 Space的列表。
- **运行控制地图**：基于 **OpenStreetMap**（Leaflet + react-leaflet），展示各 Studio 站点位置；支持点击选中、双击进入实验室；图例区分运行正常/降级/离线。
- **Studio 详情抽屉**：点击某台 Studio 后右侧滑出，展示状态概览、健康度监控、关联告警、操作（**进入实验室**、查看 Live Graph）；展示型号（M300 完整 Studio / M200 网关）；」。

### 4.3 Studio 类型与数据

- **StudioNode** 支持 `model?: "M300" | "M200"`：M300 为完整 Studio，M200 为可运行 Studio 的小型网关；Mock 中部分节点为 M300、部分为 M200。
- Studio 具备 `lat`/`lng` 用于地图打点；健康度指标含 CPU、内存、网络延迟、设备在线率、Logic 成功率、Graph 节点数等。

## 5. 工作台（/lab/）

### 5.1 入口与目标

- **直接进入工作台**（左侧点击「工作台」）：无指定 Studio，下发目标为「不指定（对所有 Studio）」；可对所有 Studio 下的设备编程。
- **从总览选中 Studio 后进入实验室**：跳转 `/lab?studio=xxx`，当前目标为该 Studio；全屏不显示壳；可「远程连接 Local Studio」（新开浏览器窗口访问该设备）。

### 5.2 Tab 结构

| Tab | 说明 |
|-----|------|
| **空间** | 构建空间本体；现场实施时可同步到本地 Studio |
| **健康度** | 单/多 Studio 运行与资源指标 |
| **逻辑设计** | 编排 → 编译 → 下发到设备 |
| **调试诊断** | 远程日志与命令 |

默认打开 **空间** Tab。

### 5.3 空间（空间建模）

- **目标**：先建模空间本体（户型、房间、设备/家具），现场实施时可同步到本地 Studio。
- **顶栏**：说明文案 + 当前目标 Studio（若有）；操作：同步、+ 新增实体、保存。
- **左侧 · 空间结构**：树形房间（客厅、主卧、厨房、卫生间等），可展开查看其下设备与家具；搜索「搜索空间/设备/家具」；筛选「全部 / 已配置图元 / 未配置图元」；底部统计「19 个对象 · 8 条关系」。
- **中间 · 组件库**：文案「拖拽图标到户型图中进行配置」；分类：家具（沙发、床、桌子、椅子、电视柜）、设备（吸顶灯、传感器、空调、窗帘）；项可拖拽。
- **右侧 · 户型视图**：图层（房间/天花板/地板）、方位关系生成、重新上传识别；户型区域按房间分块，可放置设备/家具占位。

### 5.4 健康度

- **单 Studio**：URL 带 `?studio=xxx` 时仅展示该 Studio 健康卡片。
- **多 Studio**：无 `studio` 时展示当前 Space 下所有 Studio 健康概览。
- 每张卡片：名称、型号（M300 / M200 网关）、状态（正常/降级/离线）、位置、CPU、内存、最后心跳、运行时长、Logic 成功率、Graph 节点数、网络延迟、设备在线率、固件等。

### 5.5 逻辑设计

- **未选 Studio**：左侧展示「设备」面板，按 Studio 分组列出设备，可勾选或拖拽到画布参与编排；下发时按 Studio 分组；底部入口「按需编排（轻量设备）」。
- **已选 Studio**：仅画布；画布上方有「按需编排」链接。
- **逻辑块编辑**：顶栏含备注、缩放、启用、属性；流程提示「编排 → 编译 → 下发」；**编译** 按钮（点击后视为编译通过）；**下发到设备** 按钮（仅在编译通过后可点）。
- **按需编程**：独立简页，说明不支持完整 Logic 块的设备可在 Builder 上按需编排并下发；入口「新建按需编排」。

### 5.6 调试诊断

- 展示远程日志（Mock：心跳、Logic 执行、Device、Auth、错误等）。
- 命令输入框 +「执行」按钮；有当前目标 Studio 时文案提示「当前目标：xxx」。

## 6. 资产库（/assets/）

### 6.1 Tab

- **插件广场**：浏览与使用方案、插件、驱动。
- **发布方案**：逻辑提取、从 Studio 拉取、本地上传；脱敏封装与 SAP 化。
- **我的发布**：已提交方案列表及状态（草稿/待审核/已发布/已拒绝）。
- **版本管理**：方案版本列表与回滚。

### 6.2 产品说明

- 信任资产化：将「对人的理解」转化为可订阅的数字资产（SAP）；与硬件兼容产品及订阅分成形成共利闭环。

---

## 7. 星球（/collab/）

### 7.1 信息架构

- **左侧导航**：动态、资讯、消息、Find Squad（项目小组）；以及我、点赞、评论等。
- **中间**：仅信息流，无发帖区；发帖入口在右侧。
- **右侧**：个人头像、关注/粉丝/点赞、**发帖**（点击弹窗发帖）；「为你推荐」热门话题与发现；活动/极客创作者招募等。

### 7.2 动态

- 看大家分享的信息流（发帖、方案、心得等）；筛选 Tab：推荐/关注/展示/自动化分享/驱动开发。
- 信息流为 X 风格：头像左、内容右；支持点赞、评论、收藏、分享；可展开评论输入。

### 7.3 资讯

- 行业新闻（今日 Matter、标准与产品动态）；**时间轴**展示：时间、来源、标题、摘要、分享；保持 UI 色调一致。

### 7.4 消息

- 谁创建了项目、加入了小组、分享了空间快照等；项目与小组动态列表。

### 7.5 Find Squad（项目小组）

- 按 Workspace 分组的频道列表；选中频道后为频道内消息区，支持文本、空间快照、遥测图表；输入区可插入空间快照/遥测图表。

---

## 8. 信任值（/trust/）

- **欢喜度看板**：基于用户反馈、干预次数、运行天数生成的信任分；按空间卡片展示分数、趋势、干预率、运行天数。
- **结算中心**：SAP 订阅、远程代维服务费、硬件推荐返点等类型与金额；表格展示类型、周期、金额。

---

## 9. 身份与 Space（/identity/）

- 账号与 Builder Space 管理；当前 Space、角色展示。
- 创建团队 Space、升级为公司 Space、设置团队归属企业、重置为个人等。
- 快捷入口：创建第一个项目、连接 Studio、浏览模板与方案。

---

## 10. 其他页面

- **首页（/）**：欢迎引导、步骤（认证与组织、总览、协作与资产、信任与收益）；可跳过并直接进入总览；支持 `?welcome=1` 重新显示引导。
- **个人主页（/profile/）**：最近打开的项目与 Studio 等。
- **定价（/pricing/）**、**设置（/settings/）**：占位或基础设置项。

## 11. 数据模型与 Mock

### 11.1 账号与 Space

- `context/AccountContext.tsx`：Account、BuilderSpace 列表、当前 Space ID、持久化（localStorage）；`getFirstLoginState` 默认带 `region: "CN"`。
- `lib/account-types.ts`：Account、BuilderSpace、BuilderSpaceMember、DataCenterRegion 等类型定义。

### 11.2 Studio

- `lib/studios-mock.ts`：`STUDIOS_BY_ID`，每台含 id、name、location、status、cpuLoad、memoryUsage、lat/lng、model（M300/M200）等；`getStudiosByIds`、`getAllStudioIds`。
- `components/dashboard/ClusterCard.tsx`：`StudioNode` 接口定义。

### 11.3 项目与站点

- `context/ProjectsContext.tsx`：项目、站点、Studio 关联等（与总览业务协作视角一致）。
- `lib/domain-types.ts`：BuildingType、项目/站点相关类型。

### 11.4 星球与资讯

- 星球：动态、资讯、消息、Find Squad 的 Mock 数据在 `app/collab/page.tsx` 内；资讯为时间轴结构，含 time、source、title、summary、分享。
- 资产库：`lib/plugins-mock.ts`、`lib/solution-drafts` 等。

## 12. 技术栈与工程结构

### 12.1 技术栈

- **框架**：Next.js 14（App Router）、React 18。
- **样式**：Tailwind CSS、全局 CSS 变量（如 `--bg-primary`、`--accent`、`--border-subtle`）。
- **地图**：Leaflet + react-leaflet，OpenStreetMap 瓦片；`next/dynamic` 对地图组件 `ssr: false`。

### 12.2 关键路径

| 用途 | 路径 |
|------|------|
| 根布局与条件壳 | `app/layout.tsx`、`components/ConditionalShell.tsx` |
| 顶栏与导航 | `components/AppShell.tsx` |
| 总览 | `app/spaces/page.tsx`、`components/dashboard/*` |
| 工作台 | `app/lab/page.tsx`、`components/lab/*` |
| 星球 | `app/collab/page.tsx` |
| 资产库 | `app/assets/page.tsx`、`components/extractor/*` |
| 信任值 | `app/trust/page.tsx` |
| 账号与 Space 类型 | `lib/account-types.ts` |
| 账号状态与持久化 | `context/AccountContext.tsx` |
| Studio Mock | `lib/studios-mock.ts` |

### 12.3 工作台组件

- `LabSpaceView`：空间建模（空间结构、组件库、户型视图）。
- `LabHealthView`：健康度卡片（单/多 Studio）。
- `LogicDesignView`：逻辑设计（设备面板 + 画布 / 仅画布；按需编排入口）。
- `LogicBlockEditor`：逻辑块画布（备注、缩放、启用、属性、编排→编译→下发、编译、下发到设备）。
- `LabDebugView`：调试诊断（日志、命令输入）。
