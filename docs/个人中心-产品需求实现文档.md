# 个人中心 · 产品需求实现文档

本文档基于当前工程代码整理，描述 Aqara Builder 中与「个人中心」相关的页面、入口与数据逻辑。

---

## 1. 概述

个人中心涵盖：**顶栏用户菜单**、**个人主页**、**身份与 Space**、**设置**。用户通过顶栏头像进入下拉菜单，可访问个人主页、身份与 Space、设置及工作台快捷入口；个人主页集中展示「我打开的」「我创建的」「别人分享给我的」项目与工作台；身份与 Space 负责账号与 Builder Space 的创建、切换与升级。

---

## 2. 顶栏用户菜单

### 2.1 入口

- 顶栏右侧：用户头像按钮（显示 `account.name` 首字母），点击展开下拉菜单。
- 点击菜单外区域或选择某项后关闭菜单。

### 2.2 菜单结构（自上而下）

| 区块 | 内容 |
|------|------|
| **账号** | 显示当前账号名称、邮箱；链接「个人主页」、「我的空间」；按钮「退出登录」 |
| **工作台** | 快捷链接：总览、工作台 |
| **偏好** | Appearance、Language（当前为展示项，无跳转） |
| **Settings** | 链接「设置」 |
| **支持** | Help、Send Feedback（当前为展示项，无跳转） |

### 2.3 实现位置

- `components/AppShell.tsx`：顶栏头像按钮、`userMenuOpen` 状态、下拉卡片与上述链接/按钮；图标通过 `NavIcon`（identity、settings、home、log-out、spaces、lab、credit-card、appearance、language、help、feedback）渲染。

## 3. 个人主页（/profile/）

### 3.1 页面标题与信息

- 副标题：「个人主页」；主标题：当前账号名称（`account?.name ?? "我"`）；副文：账号邮箱。

### 3.2 Tab 结构

| Tab | 说明 | 数据来源 |
|-----|------|----------|
| **我打开的** | 最近在总览或工作台打开的项目与 Studio，便于快速回到上下文 | `lib/recent.ts`：`loadRecent()`，本地存储 `builder_profile_recent`，最多 20 条 |
| **我创建的** | 当前 Builder Space 下由当前用户创建或管理的项目 | `useProjects().getProjectsByBuilderSpaceId(currentBuilderSpace.id)` |
| **别人分享给我的** | 其他 Builder 将当前用户添加为协作者的项目 | `projects` 中 `collaborators` 包含当前 `account.email` 的项目 |

每个 Tab 右侧展示数量角标（若有）。

### 3.3 我打开的

- 说明文案：「最近在总览或工作台打开的项目与 Studio 会显示在这里，便于快速回到上下文。」
- 无记录时：空状态文案「暂无最近打开记录」+ 链接「去总览打开项目 →」跳转 `/spaces/`。
- 有记录时：列表项为链接，展示 `label`，右侧小字为「项目」或「工作台」（根据 `RecentItem.type`）；点击跳转 `item.href`。

**RecentItem 结构**（`lib/recent.ts`）：

- `type`: `"project" | "lab"`
- `id`: string
- `label`: string
- `href`: string
- `openedAt`: string（ISO）

**写入时机**：总览或工作台打开某项目/Studio 时调用 `recordRecentOpened({ type, id, label, href })`，会去重并保留最近 20 条。

### 3.4 我创建的

- 说明文案：「当前 Space『{当前 Space 名称}』下由您创建或管理的项目。」
- 未选 Space：空状态「请先在顶栏选择或创建 Builder Space」。
- 当前 Space 下无项目：空状态「当前 Space 下暂无项目」+ 链接「去总览创建项目 →」。
- 有项目：列表项展示项目名称、站点数（`getSitesByProjectId(p.id).length`），点击跳转 `/spaces/`。

### 3.5 别人分享给我的

- 说明文案：「其他 Builder 将您添加为协作者的项目，您可查看或参与协作。」
- 无数据：空状态「暂无别人分享给您的项目」+ 提示「项目所有者可在编辑项目中通过『协作者』添加您的邮箱」。
- 有数据：列表项展示项目名称、当前用户在该项目中的角色（`myCollab?.role ?? "viewer"`），点击跳转 `/spaces/`。

### 3.6 实现与依赖

- 页面：`app/profile/page.tsx`
- 依赖：`useAccount()`、`useProjects()`、`loadRecent` / `RecentItem`（`lib/recent.ts`）

## 4. 身份与 Space（/identity/）

### 4.1 页面说明

- 标题：「身份与 Space」
- 副标题：「账号是唯一操作者身份；Builder Space 是平台上的经营主体（项目、资产、Studio 归属）」

### 4.2 当前状态卡片

展示只读信息：

| 字段 | 说明 |
|------|------|
| 账号（Account） | 名称、邮箱 |
| 当前 Builder Space | 名称 + 类型标签（个人/团队/公司）· 套餐 |
| 套餐 | 当前 Space 的 plan（如 free） |
| 我的角色 | Owner / Admin / Member / Viewer |
| 已绑定 Studio | 数量；若为 0 显示「去绑定」链接到 `/spaces/` |

### 4.3 快速开始（仅个人 Space 时展示）

- 条件：仅有一个 Builder Space 且其 `kind === "personal"`。
- 内容：说明「您正在使用 Personal Builder Space（Free）…」；三个快捷入口卡片：创建第一个项目（/assets/）、连接 Studio（/spaces/）、浏览模板与方案（/assets/）；底部按钮「创建 Team Builder Space →」打开创建 Team 流程。

### 4.4 我的 Builder Space 列表

- 标题「我的 Builder Space」；若有非个人 Space，右侧按钮「+ 创建 Team Builder Space」。
- 列表展示逻辑：
  - **个人 Space**：单行展示名称、类型·套餐、成员数·Studio 数；当前 Space 高亮；操作「当前」/「切换」。
  - **公司 Space**：先展示公司行（名称、企业·套餐、子 Team 数）、其下缩进展示归属该企业的 Team 子空间；每个 Team 可「改为独立」或「切换」；公司行可切换为当前。
  - **独立 Team**（无 parentCompanyId 或 parentCompanyId 为空）：单独列出；若有公司 Space，显示「归属到企业」下拉（选某公司或「独立」），以及「当前」/「切换」。

### 4.5 创建 Team Builder Space

- 弹窗多步：
  1. **步骤 1**：输入 Team 名称（placeholder：「例如：交付组、设计团队」）；若已有公司 Space，可选「归属到企业」；取消 / 下一步。
  2. **步骤 2**：选择套餐（当前为 Team Plan 说明，模拟流程无需支付）；上一步 / 确认并创建。
  3. **步骤 3**：创建成功提示；完成关闭弹窗。
- 调用：`createTeamBuilderSpace({ name, companyId?: teamUnderCompanyId })`。

### 4.6 升级为公司主体

- 展示条件：当前 Space 为 Team（`currentBuilderSpace?.kind === "team"`）且账号下尚无 Company Space。
- 卡片说明：团队公司化后可升级为 Company Builder Space（公司资料、统一成员与项目管理、CRM、企业账单、审计与治理）。
- 按钮「升级为 Company Builder Space」打开弹窗：输入公司名称，确认后调用 `upgradeToCompanyBuilderSpace({ companyName })`。

### 4.7 成长路径与其它

- 文案：「成长路径：Personal（Free）→ Team（协作/付费）→ Company（企业主体，其下可创建多个 Team 子空间）」。
- 链接「定价」到 `/pricing/`；按钮「模拟首次注册」调用 `resetToPersonal`（用于演示重置为仅个人 Space）。

### 4.8 实现与依赖

- 页面：`app/identity/page.tsx`
- 依赖：`useAccount()` 提供的 `account`、`currentBuilderSpace`、`builderSpaces`、`setCurrentBuilderSpaceId`、`myRoleInCurrentSpace`、`createTeamBuilderSpace`、`upgradeToCompanyBuilderSpace`、`setBuilderSpaceParentCompany`、`resetToPersonal`。
- 类型与文案：`KIND_LABELS`（个人/团队/公司）、`ROLE_LABELS`（Owner/Admin/Member/Viewer）。
