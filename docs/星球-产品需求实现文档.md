# 星球 · 产品需求实现文档

本文档基于当前工程代码整理，描述 Aqara Builder「星球」模块的信息架构、动态、资讯、消息、Find Squad（项目小组）、发帖与右侧栏等能力。

---

## 1. 模块概述

**星球**（路径 `/collab/`）是 Builder 的社区与协作中枢，slogan「一人一面 · 空间智能 · 共利」。左侧导航切换四个主 Tab：**动态**、**资讯**、**消息**、**Find Squad（项目小组）**；中间为主内容区（仅信息流，无发帖区）；右侧为个人卡片、发帖入口（弹窗）、为你推荐/发现与活动。发帖、点赞、评论等交互与项目小组内的空间快照/遥测图表协作均在当前页内完成。

---

## 2. 整体布局

### 2.1 三栏结构

- **左侧**：固定宽度约 220px，sticky；标题「星球」+ 副标题；主导航（动态、资讯、消息、Find Squad）；分隔后为「我」「点赞」「评论」等入口（当前为占位数字 0）。
- **中间**：主内容区，随 `planetTab` 渲染对应视图；**无发帖输入框**，发帖入口在右侧。
- **右侧**：约 300px，lg 及以上显示；sticky；个人卡片、发帖按钮、为你推荐（动态 Tab）或发现+活动（其他 Tab）。

### 2.2 主 Tab 状态

- `planetTab`：`"feed"` | `"news"` | `"messages"` | `"groups"`，对应动态、资讯、消息、Find Squad。

---

## 3. 动态 Tab（feed）

### 3.1 信息流筛选

- 顶部 Tab：推荐、关注、展示、自动化分享、驱动开发（`feedFilter`）；说明文案：「说明：中国大陆用户已隐藏仅限全球区的内容。」

### 3.2 内容区（CommunitySquare）

- 当 `planetTab === "feed"` 时传入 `hideActivityStrip: true`，即**不展示**「谁创建了项目/加入了小组」等动态条（该内容在「消息」Tab）。
- **同行分享信息流**：`MOCK_SHARES`（SharePost 列表）；布局为头像左、内容右（类 X/微博）。
- 每条帖子：作者名、标签（方案/踩坑/心得/问答/其他）、时间、正文、可选配图（1～4 张）；操作：评论（展开评论列表+输入）、点赞（持久化到 localStorage）、收藏数、分享数。
- **点赞**：`STORAGE_PLANET_LIKES` 存储已点赞 post id 列表；toggle 时更新并 `saveLikedShareIds`。
- **评论**：`STORAGE_PLANET_COMMENTS` 存储按 postId 的评论列表；新增评论写入「我」+ 内容 +「刚刚」；`addComment` 后更新 `commentsByPostId` 并清空该条输入。

### 3.3 SharePost 类型

- id、authorId、authorName、avatar、tag、content、images?、ts、likeCount、commentCount?、saveCount?、shareCount?。

## 4. 资讯 Tab（news）

### 4.1 标题

- 「今日资讯」；副标题：「行业新闻 · Matter / 标准 / 产品」。

### 4.2 时间轴

- 左侧虚线竖线 + 每条左侧圆点；右侧为资讯卡片。
- **NewsItem**：id、tag、source、time、title、summary、date、href。
- 卡片内容：tag（如「今日 Matter」）、来源、时间、标题、摘要、链接/分享；风格与整体 UI 一致。

### 4.3 Mock（MOCK_NEWS）

- 示例：Matter 1.2 发布、空间训练师 Meetup、Thread 1.3 与 Matter 协同、Builder 空间治理能力 2.0 等。

---

## 5. 消息 Tab（messages）

### 5.1 标题与说明

- 「消息」；副标题：「项目与小组动态」。

### 5.2 内容（CommunityMessages）

- 展示「谁创建了项目、加入了小组、分享了空间快照」等动态列表（`MOCK_ACTIVITY` 等）。
- 每条：头像、谁、动作、对象、时间（如「2 小时前」「昨天」）。

---

## 6. Find Squad Tab（项目小组，groups）

### 6.1 布局

- 左侧：**频道列表**（按 Workspace 分组）；右侧：**当前频道** 的头部 + 消息列表 + 输入区。

### 6.2 频道列表

- `channelsByWorkspace`：来自 `MOCK_WORKSPACES` 与 `MOCK_CHANNELS`；每个 Workspace 下列出所属 Channel。
- 每个频道按钮展示：名称、未读数（若有）、最后一条消息预览、成员数 · 时间。
- 点击频道设置 `activeChannel`；无选中时右侧提示「从左侧选择项目频道开始协作」。

### 6.3 频道头部

- 频道名称、描述、成员数、成员头像（前 4 个）、「工作台」链接（/lab/）。

### 6.4 消息列表

- `channelMessages`：从 `MOCK_MESSAGES` 中筛选 `msg.channelId === activeChannel`。
- **消息类型**：text（纯文本）、space_snapshot（空间快照）、telemetry_chart（遥测图表）。
- **MessageItem**：头像、作者名、角色、时间；根据类型渲染：
  - **文本**：MessageBubbleText。
  - **空间快照**：MessageBubbleSpaceSnapshot（Studio、空间名、空间数、快照时间、占位缩略图、「在 Studio 工作台查看」链接到 `/lab?studio=xxx`）。
  - **遥测图表**：MessageBubbleTelemetryChart（Studio、指标、时间范围、柱状图、「查看详情/时空回溯」链接）。
- 相邻同作者且同为 text 时可合并显示作者行（showAuthor 逻辑）。

### 6.5 输入区

- **待发送附件**：若有「待插入空间快照」或「待插入遥测图表」，在输入区上方显示标签，可移除。
- **插入入口**：「插入空间快照」「插入遥测图表」按钮 → 打开 `InsertPicker` 弹层。
- **InsertPicker**：选择 Studio、空间名称（快照时可选）、指标与时间范围（遥测时）；确认后写入 `pendingSnapshot` 或 `pendingChart`，发送时随消息一起提交（当前为前端状态，无后端）。
- **输入框**：placeholder「输入消息… 支持 @提及成员（输入 @ 触发）」；Enter 发送；**发送**按钮（无内容且无附件时禁用）。

### 6.6 Channel / Message 类型

- **Channel**：id、workspaceId、workspaceName、name、description、unread、lastMsg、lastTs、memberCount、members。
- **Message**：id、channelId、authorId、authorName、authorRole、ts、type、body?、spaceSnapshot?、telemetryChart?。

## 7. 右侧栏

### 7.1 个人卡片

- 当前账号头像（首字母）、名称、邮箱；关注/粉丝/点赞数（Mock：864、72、1056）；**发帖**按钮 → `setShowPostModal(true)`。

### 7.2 发帖弹窗

- 标题「发帖」；关闭按钮；当前用户头像 + 多行输入框（placeholder「有什么想分享的？」，当前为 readOnly 占位）；底部：图片、空间快照图标、「发布」按钮（点击关闭弹窗）。发布逻辑为占位，未接入真实发帖 API。

### 7.3 动态 Tab 下的右侧

- 「为你推荐」标题 +「热门话题与发现」；`MOCK_HOT_TOPICS` 列表（tag + 数量）；「极客创作者招募」活动卡片 +「查看活动」按钮。

### 7.4 非动态 Tab 下的右侧

- 「发现」+ 热门话题列表（同 MOCK_HOT_TOPICS）；「活动」标题 + `MOCK_ACTIVITIES` 卡片列表（活动/挑战等）。

---

## 8. 数据与存储

### 8.1 Mock 与类型

- **Workspace / Channel / Studio**：MOCK_WORKSPACES、MOCK_CHANNELS、MOCK_STUDIOS（与空间站/资产库拉取等对齐）。
- **资讯**：MOCK_NEWS（NewsItem[]）。
- **动态**：MOCK_ACTIVITY（项目/小组/快照类动态）；MOCK_SHARES（SharePost[]）；MOCK_SHARE_COMMENTS（Record<postId, ShareComment[]>）。
- **消息**：MOCK_MESSAGES（Message[]）；SpaceSnapshotPayload、TelemetryChartPayload。

### 8.2 本地持久化

- `STORAGE_PLANET_LIKES`：已点赞的 post id 数组。
- `STORAGE_PLANET_COMMENTS`：按 postId 的评论列表（用户新增评论会合并写入）。
