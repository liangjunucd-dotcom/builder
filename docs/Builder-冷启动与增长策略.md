# Builder 冷启动与增长策略

> 核心命题：如何让全球 Builder 在 30 秒内「玩起来」，并自发传播？

---

## 一、Slogan

### 主 Slogan

**"AI Build — 一句话，造空间"**

英文版: **"AI Build — Describe it. Build it."**

### 副 Slogan（场景化）

| 场景 | Slogan |
|------|--------|
| 自动化 | "一句话编程，让空间自己运转" |
| 组态大屏 | "说出来，控制面板就出来" |
| Studio Live | "对话真实空间，灯真的会亮" |
| 传播用 | "我用一句话造了一个智能空间" |

## 二、冷启动路径：三步走

### Phase 0：Playground（当前优先）

**目标**：让开发者 30 秒内获得"哇"的体验，无门槛。

```
用户来到 Builder
    ↓ 无需登录即可尝试
输入一句话（如"帮我打开客厅的灯"）
    ↓ AI 即时响应
看到效果（自动化流程/组态大屏/真实灯亮了）
    ↓ 想保存或分享
登录/注册 → 保存到 My Builds
    ↓ 分享到社区
其他用户看到 → 也来玩
```

**三种 Build 模式**：

| 模式 | 输入 | 输出 | 传播性 |
|------|------|------|--------|
| **Build Automation** | "有人进客厅开灯，5分钟没人关灯" | 可视化自动化流程图 | 中 |
| **Build Dashboard** | "给我一个三室两厅的控制大屏" | 可交互组态面板 | 高 |
| **Studio Live** | "帮我打开灯" / "查询设备列表" | 真实空间视频 + 实时效果 | 极高 |

### Phase 1：社区 & 沉淀（3-6个月）

用户创作内容沉淀为可复用资产：
- 自动化脚本可被 Fork / 复用
- 组态大屏模板可被分享 / 交易
- 社区排行榜、热门创作

### Phase 2：项目 & 商业化（6-12个月）

Builder 有沉淀后，自然延伸：
- 方案模板 → 项目管理
- Installer 工具链
- 客户管理 & 服务闭环

## 三、Studio Live — 杀手级体验

### 架构

```
Builder 云端
├── OpenClaw AI Agent（每用户独立 Session）
├── 模拟 Studio（无需排队，默认模式）
│   └── 虚拟设备状态 + 模拟响应
└── 真实 Studio（排队模式）
    ├── 物理空间中的 Aqara Hub + 设备
    ├── 摄像头视频流（WebRTC/HLS）
    └── AI → Studio API → 设备动作 → 视频可见
```

### 排队机制

- 默认进入「模拟模式」，零等待
- 用户可点击「切换到真实 Studio」
- 真实 Studio 采用排队制：显示前方人数 + 预计等待时间
- 每人最长互动 3 分钟，超时自动释放
- 排队时可继续在模拟模式中探索

### 传播价值

想象一个开发者发了条推文：
> "我刚在 Aqara Builder 上说了一句话，远程真的把灯打开了 🤯 试试看 → [链接]"

这种「真实效果」是纯软件 Demo 无法比拟的传播力。

## 四、模式可行性分析

### 成本控制

| 资源 | 方案 | 成本 |
|------|------|------|
| OpenClaw AI | 1 台云端实例，多用户 Session 共享 | 低（单机可承载 ~100 并发 Session） |
| 模拟 Studio | 纯软件，无硬件成本 | 几乎为零 |
| 真实 Studio | 1-2 套物理空间（深圳/北京），排队共享 | 固定投入，边际成本低 |
| 视频流 | WebRTC P2P 或 HLS（少量并发） | 低 |

### 增长飞轮

```
低门槛体验（30秒上手）
    ↓
分享传播（社交裂变）
    ↓
社区内容积累
    ↓
开发者沉淀（注册、保存、协作）
    ↓
自然转化为付费用户（Pro/Team）
    ↓
Installer/服务商需求 → Phase 2 项目管理
```

### 竞品差异

没有任何 IoT 平台做到：
1. **一句话生成自动化**（Home Assistant 需要写 YAML）
2. **一句话生成控制大屏**（完全原创）
3. **远程对话控制真实空间 + 看到视频**（破次元壁）

这就是 Builder 的「只此一家」护城河。

## 五、导航架构调整

### 从「工具优先」到「体验优先」

**旧架构**（工具优先，重）：
```
首页 → 广场 → 创作 → 项目 → 资产
```

**新架构**（体验优先，轻）：
```
核心：
  AI Build（Playground，首页即入口）
  Studio Live（真实空间对话）
  Explore（社区画廊 + 模板）

创作：
  My Builds（我的创作）
  Projects（项目管理 — 未来扩展，低优先级）
  Studios（我的 Studio）

学习：
  Academy / Community / Forum

管理：（收进用户菜单）
  Workspace / Members / Settings
```

### 设计原则

1. **首页即 Playground**：用户打开就能输入一句话开始创作
2. **社区即内容**：首页下方展示精选创作，激发灵感
3. **Studio Live 有独立入口**：这是杀手级功能，值得高曝光
4. **项目管理低调存在**：未来可扩展，当前不是主推
5. **紫色基调**：科技感、AI 感、与 Aqara 蓝形成品牌延伸

## 六、未来可扩展性

当前设计确保以下能力可在未来无缝接入：

| 未来功能 | 扩展方式 |
|---------|---------|
| Installer 项目管理 | Projects 页面已存在，只需在菜单中提升优先级 |
| 客户管理 CRM | 已有 Customers 路由，激活即可 |
| 服务枢纽 | 已有 Service 路由，激活即可 |
| 方案模板市场 | My Builds → 发布到市场 |
| 插件生态 | 已有 Square（广场），直接扩展 |
| 企业认证 | Workspace 设置中扩展 |

**总结**：冷启动聚焦「让 Builder 玩起来」，但架构留好了「让 Builder 做生意」的所有接口。
